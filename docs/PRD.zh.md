# 催收语音 Agent — 产品需求文档(PRD)

> 一个 Demo 网页应用:**用户扮演债务人**,**AI Agent 扮演催收员**。
> 固定技术约束:**Fish Audio TTS**、**LiveKit**(实时音频)、Web 应用。
> 本文档只关注**产品范围**。技术方案见后续 `TECH_DESIGN.md`。

---

## 1. 目标与非目标

### 1.1 目标
以最短时间交付一个可运行的 Demo:通过一通语音通话驱动一个明确的业务结果——
**Agent 让用户给出一个还款承诺(金额 + 日期)**,并在通话结束后清晰展示这个结果。

### 1.2 成功标准(Demo 级别)
- 访客无需说明就能完成完整通话(开始 → 对话 → 结束)。
- 通话结束后展示**结构化总结**(是否达成承诺、金额、日期、结束原因)。
- 端到端语音延迟接近自然对话(目标:用户说完后 Agent 首字 < 2 秒)。
- Agent 始终保持人设,且能处理至少 4 类核心债务人响应(见 §6)。
- 可通过公开 URL 直接分享,无需安装和注册。

### 1.3 非目标
- 不是生产级催收系统(无真实 PII、无真实电话、无 CRM 对接)。
- 不接 SIP / 不打真实电话,只用 Web WebRTC。
- 无多租户、无登录鉴权、无持久化数据库。
- 不支持多案件切换、无管理后台。
- MVP 阶段不做高级打断策略,只做基本的轮次切换(turn-taking)。
- 不优先适配移动端(主目标:桌面 Chrome)。

---

## 2. 目标受众
- **主要受众**:技术评审 / 招聘方 / 评估语音 AI 能力的客户。
- **次要受众**:决定是否产品化的内部干系人。

含义:优先做到"60 秒内让人惊艳",而不是深度可配置。

---

## 3. 核心场景与用户故事

### 3.1 场景
> 访客打开页面,看到一个简短的案件卡片(债务人姓名、欠款金额、逾期天数、最低还款额)。
> 点击 **开始通话**,AI 催收员开口说明来意,然后引导对话朝"明确还款承诺"推进。
> 用户即兴扮演债务人。当达成承诺、用户挂断、或超时时,通话结束。
> 然后展示一张总结卡片。

**一通通话 = 一次 Demo 会话。** 无登录、无历史、不持久化。

### 3.2 单一核心用户故事
作为扮演"债务人"的 Demo 用户,我希望在一次模拟通话中被 Agent 引导给出明确还款承诺,
从而直观感受 Agent 推动业务结果的能力。

---

## 4. 分阶段功能规划

三个阶段,每个阶段独立可交付、独立可演示。

| 阶段 | 主题 | 目标 |
|---|---|---|
| **P0 — MVP** | "能跑通" | 真实端到端语音通话。可以丑,但要闭环。 |
| **P1 — 打磨** | "像真电话" | 看着像、听着像真实电话,可对外演示。 |
| **P2 — 进阶** | "令人惊艳" | 区别于通用语音 Demo 的差异化功能。 |

## 5. Phase 0 — MVP(必做)

**完成定义:** 用户能加载页面、开始通话、进行约 2 分钟的连贯对话,通话结束后展示
**结构化承诺总结**。这是证明产品可行的最小闭环。

### 5.1 功能列表

| ID | 功能 | 说明 |
|---|---|---|
| F0.1 | 案件卡片 | 通话前面板:债务人姓名、欠款总额、逾期天数、最低还款额。 |
| F0.2 | 开始/结束按钮 | 单一主按钮,在"开始通话"和"结束通话"间切换。 |
| F0.3 | 麦克风权限流程 | 标准浏览器授权弹窗;拒绝时给清晰的错误提示。 |
| F0.4 | 实时音频收发 | LiveKit 房间打通用户麦克风 ↔ Agent 语音。 |
| F0.5 | Agent 主动开口 | 连接成功后,Agent 主动给开场白说明来意。 |
| F0.6 | 轮次切换 | 基于 VAD 的简单 turn-taking,不支持打断。 |
| F0.7 | 实时转录 | 双栏滚动转录(用户 vs Agent),通话中可见。 |
| F0.8 | 通话状态与计时 | 可见状态:`空闲 / 连接中 / 通话中 / 结束中 / 已结束` + 通话时长。 |
| F0.9 | 承诺识别 | LLM 从对话中抽取 `{金额, 日期}`,在用户给出明确承诺时触发。 |
| F0.10 | 结构化总结卡片 | 通话结束后展示:是否达成承诺、金额、日期、结束原因、时长。 |
| F0.11 | 单一人设 + 单一档案 | 写死的"专业型"催收员 + 一份虚构债务人档案。 |

### 5.2 对话策略(MVP 最小可行)
Agent 目标(单一明确):**收集还款承诺 = 金额 + 日期**。

Agent 必须处理这 4 类响应:
- **A. 直接同意还款** — 确认金额和日期,复述,然后收尾。
- **B. 申请延期 / 部分还款** — 提供最低还款方案,要求一个具体日期。
- **C. 否认 / 不清楚债务** — 简短陈述账单事实,继续推进还款方案。
- **D. 拒绝交流 / 沉默** — 一次简短挽留;若仍无回应,礼貌结束。

对话原则:
- 简短、克制、合规;不威胁、不羞辱。
- 一轮一个问题;避免长篇大论。

### 5.3 结束原因(P0)
总结卡片必须明确报告以下其中一种:
- `commitment_reached` — Agent 抽取到 `{金额, 日期}` 并已确认。
- `user_hangup` — 用户点击了结束通话。
- `timeout` — 通话超过 5 分钟,或连续静默 30 秒。
- `error` — 连接 / 服务故障。

### 5.4 P0 明确不做(留给后续阶段)
- 没有"来电响铃"UI(留到 P1,P0 用普通"开始通话"按钮即可)。
- 没有打断 / 抢话(barge-in)。
- 没有人设切换、场景切换、可编辑档案。
- 没有音效、头像动画、情绪指示。
- 没有多语言。
- 不要求适配移动端。

### 5.5 P0 验收测试
进行一通约 90 秒的通话,用户分别尝试 A、B、C 三类响应。
对话保持连贯且不偏题;在 A 类中 Agent 应达成承诺;点击"结束通话"后,
总结卡片显示正确的结束原因,且(对 A 类)正确显示抽取出的金额和日期。

## 6. Phase 1 — 打磨(可对外演示)

**完成定义:** 整个 Demo 给人的感受像真实的电话通话,可放心公开分享。

### 6.1 功能列表

| ID | 功能 | 说明 |
|---|---|---|
| F1.1 | 来电 UI | 用全屏"未知号码来电" + 振铃音 + 绿色接听 / 红色拒接,替换 P0 的普通按钮。 |
| F1.2 | 通话中页面优化 | 头像、来电方姓名、静音按钮,精修过的计时和状态徽标。 |
| F1.3 | 语音活动指示 | Agent 说话时和麦克风检测到声音时显示脉冲动画。 |
| F1.4 | 打断(Barge-in) | 用户开口时 Agent 立即停止说话。 |
| F1.5 | 扩展响应处理 | 新增:愤怒辱骂、卖惨、试图提前挂断、"你是不是 AI?"。 |
| F1.6 | 加强版总结卡片 | 新增:关键要点列表、对承诺内容的复述。 |
| F1.7 | 连接错误状态 | 麦克风被拒 / 网络断开 / Agent 不可用,各有清晰 UI。 |
| F1.8 | 加载/连接反馈 | 骨架屏 + 振铃过渡音,死气时间 ≤ 1 秒。 |

### 6.2 质量基线(P1)
- 点击接听后到 Agent 首字播出:**< 2.5 秒**。
- 用户说完后 Agent 首字播出:**< 1.8 秒**(目标),< 2.5 秒(可接受)。
- 字幕在说话后 500ms 内出现。
- 3 分钟通话内零音频卡顿。

### 6.3 P1 明确不做(留给 P2)
- 不做人设切换、场景切换。
- 不做情绪可视化。
- 不做多语言。
- 不做埋点统计。

### 6.4 P1 验收测试
一位非技术观察者在没有任何前置说明的情况下,看完一通 3 分钟的演示通话后,
说出"感觉像真打电话一样"。字幕、总结、挂断流程全部正常。

## 7. Phase 2 — 进阶(差异化亮点)

**完成定义:** Demo 至少包含 3 个 P2 功能,使其明显区别于通用语音 AI 样例。

按"投入产出比"排序,挑选时间允许范围内的功能即可:

### 7.1 功能候选清单(按性价比排序)

| ID | 功能 | 为什么惊艳 | 投入 |
|---|---|---|---|
| F2.1 | **人设切换**(专业型 / 共情型 / 强硬型) | 展示 TTS 的语调范围 + prompt 设计。 | 低 |
| F2.2 | **场景预设**(首次电催 / 二次提醒 / 承诺违约) | 展示 Agent 能根据上下文调整。 | 低 |
| F2.3 | **可编辑债务人档案**(姓名、金额、到期日) | 让评审现场测试边界场景。 | 低 |
| F2.4 | **Agent 情绪指示**(头像情绪:平静/严肃/共情) | 可视化 Agent 当前"状态"。 | 中 |
| F2.5 | **结构化通话结果**(JSON 视图:意图分类、承诺日期、风险评分) | 展示 LLM 工具调用 / 信息抽取能力。 | 中 |
| F2.6 | **音频波形可视化** | 现代语音 UI 的基本预期。 | 低 |
| F2.7 | **多语言**(英文 + 中文,首页切换) | 扩大受众,展示 TTS 灵活性。 | 中 |
| F2.8 | **合规护栏演示**(尝试让 Agent 威胁 → 它拒绝) | 展现安全设计思考。 | 低 |
| F2.9 | **重放上一通通话**(下载音频 + 转录) | 方便评审分享。 | 中 |
| F2.10 | **"幕后面板"**(可切换:实时显示 STT 文本、LLM prompt、TTS 分块) | 对技术评审极具吸引力。 | 中 |

### 7.2 推荐的 P2 最小集(3 个)
**F2.1(人设切换) + F2.2(场景预设) + F2.10(幕后面板)**

理由:人设 + 场景展现产品广度;幕后面板对本 Demo 的*技术受众*尤其有杀伤力。

## 8. 债务人响应分类(完整参考)

P0 收敛为 4 类核心(A–D)。P1 / P2 增加更细粒度变体。

| 类别 | 用户示例 | Agent 策略 | 阶段 |
|---|---|---|---|
| **A. 直接同意** | "我下周五还 500。" | 确认并复述金额和日期;收尾。 | P0 |
| **B. 申请延期 / 部分还款** | "下个月只能还最低额可以吗?" | 提供最低还款方案,要求具体日期。 | P0 |
| **C. 否认 / 不清楚** | "我没借过这笔钱。" | 引用合同号 + 卡尾号;引导回还款方案。 | P0 |
| **D. 拒绝 / 沉默** | (沉默)或"别再打了。" | 一次简短挽留;无回应则结束。 | P0 |
| 哭穷 / 卖惨 | "我刚失业了……" | 表达理解;提供分期选项。 | P1 |
| 愤怒 / 辱骂 | "别再骚扰我!" | 保持冷静;放慢语速;尝试拉回一次,否则礼貌结束。 | P1 |
| 试图提前挂断 | "我没空,挂了。" | 一句简短重申后果,留下回拨方式。 | P1 |
| 质疑你是 AI | "你是机器人吧?" | 如实承认是智能助理。 | P1 |
| 第三方接听 | "他不在。" | **不**透露债务详情;请其转告回拨。 | P2 |
| 要求转人工 | "给我转真人。" | 表示理解;提供回拨号码。 | P2 |

## 9. 人设目录

| 人设 | 适用场景 | 声音特征 | Prompt 特征 |
|---|---|---|---|
| **专业型(P0 默认)** | 标准催收 | 语速均匀、中性温度 | 礼貌、精确、合规、不施压 |
| **共情型(P2)** | 困难户 | 语速更慢、更柔和 | 多用"我理解",提供选项,降低紧迫感 |
| **强硬型(P2)** | 多次违约 | 语速略快、音调略低 | 明确说明后果,但仍合规 |

> 注意:"强硬型"绝不能跨越到威胁。合规约束对所有人设一视同仁。

## 10. 场景预设(P2)

| 预设 | 债务人状态 | Agent 目标 |
|---|---|---|
| 首次接触 | 逾期 12 天,无前置沟通 | 确认本人,告知事实,获取还款日期 |
| 二次提醒 | 逾期 30 天,1 次前置通话 | 引用上次通话,提升紧迫度 |
| 承诺违约 | 错过了之前承诺的日期 | 引用承诺,询问原因,重新协商 |

## 12. 状态机

前后端共用的唯一权威定义。

状态:
- `IDLE` — 页面已加载,尚未开始。
- `CONNECTING` — 加入 LiveKit 房间,Agent 启动中。
- `IN_CALL_AGENT_TURN` — Agent 正在说话。
- `IN_CALL_USER_TURN` — 用户正在说话 / 房间空闲等待用户。
- `ENDING` — 生成总结(抽取金额日期、判定结果)。
- `ENDED` — 总结已展示。
- `ERROR` — 连接 / 服务故障。

转移:
- `IDLE → CONNECTING`:用户点击 开始通话(P0)/ 接听(P1)。
- `CONNECTING → IN_CALL_AGENT_TURN`:房间连通,Agent 开口。
- `IN_CALL_AGENT_TURN ↔ IN_CALL_USER_TURN`:由 VAD 驱动的轮次切换。
- `* → ENDING`:用户结束通话,或承诺被识别并确认,或超时触发。
- `ENDING → ENDED`:总结生成完毕。
- `* → ERROR`:任何不可恢复故障。

## 13. 边缘情况与失败模式

| 情况 | 处理方式 |
|---|---|
| 麦克风被拒 | 阻止开始通话;给一行修复指引。 |
| 通话中断网 | LiveKit 自动重连(≤5 秒);顶部横幅提示;失败则平稳结束 + 总结。 |
| Agent 后端不可用 | 禁用接听按钮;提示"Agent 离线"。 |
| 用户长时间不说话 | 静默 15 秒后 Agent 主动询问;30 秒后礼貌结束。 |
| ASR 听错金额/日期 | Agent 在确认前主动复述。 |
| 浏览器音频自动播放被拦截 | 仅在用户点击后播放(接听按钮即可解锁)。 |
| 用户辱骂 / 试图越狱 | system prompt 硬约束 + 输出过滤;Agent 保持人设。 |
| 通话超过 5 分钟 | Agent 优雅收尾。 |

## 14. 合规与伦理约束(贯穿所有阶段)

Agent **不允许**:
- 虚假威胁法律后果或暗示暴力。
- 向非债务人本人透露债务详情。
- 不论被如何挑衅都不得使用辱骂语言。
- 在被直接询问时声称自己是真人。
- 在 8:00 前 / 21:00 后呼出(Demo 不强制,但 prompt 要写明)。

Agent **必须**:
- 在通话开始时表明自己和公司身份。
- 允许债务人随时结束通话。
- 在结束时提供回拨联系方式。

这些规则同时落在 system prompt 和输出过滤器中。
